常用命令 - Linux命令

grep命令

grep（Global Regular Expression Print）使用正则表达式搜索文本，并把匹配的行打印出来。

  • 查找文件file.log中“password”字段：grep ”password” file.log
  • 查找文件file.log中“passport”字段，并且统计出出现次数：grep “password” file.log | wc -l 或者 grep “password” file.log -c

管道操作符“|”仅能处理经由前面一个指令传出的正确输出信息，也就是standard output的信息，对于standard error信息没有直接处理能力。然后传递给下一个命令，作为标准的输入standard input。然后wc(word count)可以统计指定文件中的字节数，字数，行数，并将统计结果显示输出。

top和ps命令：探测进程

ps命令

默认只会显示运行在当前控制台下的属于当前用户的进程

  • ps -A和ps -e可以显示所有进程
  • ps -ef显示完整格式的所有进程
  • 指定进程名，ps -ef | grep “java” 找出进程名中包括java的所有进程

top命令

可以实时监控进程，输出的第一部分是显示系统的概括。

  • 第一行显示了当前时间、系统的运行时间、登录的用户和系统的平均负载（1min 5min 15min）
  • 第二行显示了进程的概要信息，有多少进程处于运行、休眠、停止或者僵化状态
  • 第三行是CPU概要信息
  • 第四行是系统内存的状态

ps和top的区别：

  • ps看到的是命令执行瞬间的进程信息，而top可以持续的监控
  • ps只是查看进程，而top还可以监视系统性能，如平均负载，cpu和内存的消耗
  • top可以操作进程，如改变优先级（命令r）和关闭进程（命令k）
  • ps主要是查看进程的，关注点在于查看需要查看的进程
  • top主要看cpu，内存使用情况，及占用资源最多的进程由高到低排序，关注点在于资源占用情况。

sed命令

sed命令是利用脚本来处理文本文件。sed可按照脚本的指令来处理、编辑文本文件。sed主要用来自动编辑一个或多个文件、简化对文件的反复操作、编写转换程序等。

sort命令

  • 正序排序：sort -n test.txt
  • 反序排序：sort -nr test.txt

tail和head命令

  • tail -n 2 file.log 可以查看文件的最后两行
  • tail -f file.log 可以实时查看文件的后边追加的部分
  • head -n 2 file.log 可以查看文件的开始2行

Maven命令

Maven是什么

Maven是一个跨平台的强大构建工具，可以实现自动化构建过程，从“清理、编译、测试、生成报告、打包和部署”都可以使用成熟的插件，通过简单的命令实现，避免了重复的构建过程。

Maven不仅是一个优秀的项目构建工具，还是一个依赖管理工具，他提供了强大的中央仓库，能够帮助我们自动下载依赖。

Maven提供了一个很好的解决方案。Maven通过使用GroupId和ArtifactId来标识每一个构件（依赖），也就是通过一个坐标系精确地定位到了一个Java类库。Maven的中央仓库几乎可以找到任何流行的开源类库，通过在项目POM文件中配置，都可以免费下载。

Maven仓库

  • 本地仓库：就是Maven在本地存储构件的仓库，默认是在用户的.m2/repository/目录下
  • 远程仓库：分为中央仓库、私服以及其他公共库。用户是在和私服打交道，包括上传和下载构件。当私服满足不了我们的需求时，私服会和中央仓库或者其余公共仓库交互，将用户需要的构件缓存在私服仓库中。

私服：私服是一种特殊的远程仓库，是在局域网内的仓库服务，私服代理广域网上的远程仓库，供局域网内的Maven用户使用。

setting.xml文件

  • localRepository：本地仓库的目录。默认是用户目录下的 .m2/repository 目录。
  • interactiveMode：表示是否使用交互模式，默认为true；如果设为false，那么当Maven需要用户数据的时候，它会使用默认值
  • offline：表示是否离线，默认为false。这个属性表示Maven进行项目编译和部署等操作时，是否允许Maven进行联网来下载信息等。
  • mirrors：定义一系列的远程仓库镜像，用于缓解远程仓库的压力
  • profiles：用于指定一系列的profile
  • activeProfiles：指定当前正在活跃的profile
  • servers：表示当需要连接到一个远程服务器的时候需要使用到的验证方式

Maven的生命周期

clean生命周期

clean生命周期的目的是清理项目，删除前一次构建在target文件夹下生成的各个JAR包等，它包含三个阶段：

  • pre-clean 执行一些清理前需要完成的工作
  • clean 清理上一次构建生成的文件
  • post-clean 执行一些清理后需要完成的工作

site生命周期

site生命周期的目的是建立和发布项目站点，Maven可以给与POM所包含的信息，生成一个站点，方便团队交流和发布项目信息，其生命周期阶段包含：

  • pre- site
  • site 生成项目站点文档
  • post-site
  • site-deploy 将生成的项目站点发布到服务器上

default生命周期

default生命周期定义了真正构建项目中需要执行的所有步骤

  • validate
  • initialize
  • generate-sources
  • process-sources
  • generate-resources
  • process-resources
  • compile：编译项目的主源码
  • process-classes
  • generate-test-sources
  • process-test-sources
  • generate-test-resources
  • process-test-resources
  • test-compile
  • process-test-classed
  • test：使用单元测试框架运行测试，测试代码不会被打包或部署
  • prepare-package
  • package：接受编译好的代码，打包成可发布的格式，JAR/WAR 等
  • pre-integration-test
  • integration-test
  • post-integration-test
  • verify
  • install：安装构件到本地仓库
  • deploy：发布构件到远程仓库

常用构建命令

  • mvn clean：删除上次打包生成的target文件夹
  • mvn compile：对源代码进行编译
  • mvn clean package：用于清除上次打包生成的target文件夹并且对项目进行打包
  • mvn clean install：用于清除上次打包生成的target文件夹并且重新将项目进行打包，安装到本地仓库
  • mvn clean deploy：用于清除上次打包生成的target文件夹并且重新将项目进行打包，发布到远程仓库（私服）

Git命令

Git的工作原理

  • 工作区：也就是在我们本地的一个git仓库目录下，除了隐藏的.git文件之外的所有内容
  • 暂缓区：数据暂时存放的区域，可以在工作区和版本库之间进行数据交互
  • 版本库：就是我们的Git本地版本仓库，也就是.git目录下就是版本库。通过git push命令可以将本地的版本推送到远程仓库。
  • 远程仓库：是我们在远程进行版本控制的仓库，比如GitLab或者GitHub等服务器上的仓库。

基本命令

  • git init：初始化本地仓库
  • git add：往暂缓区中添加内容
  • git status：查看当前的状态
  • git commit -m “xxx”：提交到版本库
  • git remote add origin “xxx”：将本地仓库和远程仓库建立连接
  • git push origin master:master：将本地版本库中的文件推送到远程仓库中。
  • git branch new：创建一个新分支
  • git branch -d new：删除new分支
  • git check new：切换到新分支
  • git pull origin master：将远程master代码更新到本地当前分支
  • git merge new：将本地new分支代码合并到当前分支上

git reset –hard 和 –soft 的区别是啥

通过git log可以查看当前分支的commit_id提交记录。当我们想回退到某个版本的时候，可以采用git reset命令，之后通过git push origin branchName -f命令来强制将当前分支push到远程仓库。

  • –hard 表示将文件恢复到指定的版本，删除掉了所有的改动，包括工作区和暂缓区
  • –soft 则是将当前分支的HEAD指向指定的版本，并且把指定版本到重置之前的所有改动放入暂缓区中，经常用来合并多个commit_id

git reset 和 git revert的区别是啥

git reset是通过回退HEAD来完成版本回退。git revert是生成了一次新的提交，新的提交和你想去掉的那次提交的内容刚好相反，本质上也是一次新的提交，也就是HEAD继续向前进。
